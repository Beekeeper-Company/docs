# Введение
В этом разделе (а пока только в этой заметке) будут изложены все подробности о структуре сервера, его компонентах, их взаимодействия, возможные трудности, а также декомпозиция задач

## Архитектурный обзор
Сервер состоит из двух компонентов, обеспечивающих полноценную кооперативную игру: сервер синхронизации игрового процесса и система коммуникации между игроками. Сервер разбит на два компонента, потому что они будут реализованы поверх двух параллельно рабочих транспортных протоколов.

**UDP-компонент** отвечает за синхронизацию игрового состояния, поскольку протокол обеспечивает минимальные задержки. Здесь обрабатываются перемещения игроков, действия, физика и все изменения игрового мира. 

**TCP-компонент** возьмет на себя задачу коммуникации пользователей, а именно чат (in progress)

## Идентификация и авторизация
Задачи авторизации и управления лобби будут делегированы на **Steam SDK**, что избавляет от необходимости реализовывать собственные механизмы регистрации, управления группами и удалённого игрового сервера (поскольку создатель лобби является хостом)
## Управление сессиями
Каждое подключение к UDP-серверу представлено отдельной **сессией**, привязанной к Steam ID игрока. Сессии отслеживаются через механизм heartbeat (периодические подтверждения активности) для мониторинга состояния клиентов.
## Оптимизация трафика
Для эффективного использования пропускной способности сети применяются:
- **Interest management** — игрок получает обновления только о тех объектах, которые находятся в зоне его видимости    
- **Diff-based обновления** — отправляются только изменения состояния, а не полные снапшоты

| Компонент             | Технология                         |
| --------------------- | ---------------------------------- |
| Язык программирования | Elixir                             |
| Платформа             | OTP (Open Telecom Platform)        |
| Транспорт UDP         | `:gen_udp`                         |
| Транспорт TCP         | `:gen_tcp`                         |
| Сериализация          | JSON (Jason)                       |
| Хранение данных       | ETS (in-memory), PostgreSQL (Ecto) |
| Интеграция            | Steam SDK (NIF)                    |
| Метрики               | `:telemetry`                       |
